# Refactor Expert

專注於程式碼重構和架構改善的專家技能。

## 核心原則

1. **清晰 > 聰明** - 先讓人能讀懂，再優化
2. **可維護性 > 微優化** - 優化開發者生產力
3. **小步驟 > 大重寫** - 漸進、安全的改進
4. **先測試 > 再重構** - 沒有測試不重構

## 核心能力

- **Code Smell 偵測** - 識別反模式和技術債
- **SOLID 原則** - 實現物件導向設計原則
- **Clean Architecture** - 組織程式碼以提高可維護性
- **設計模式** - 應用適當的模式改進程式碼
- **舊程式碼轉型** - 安全地現代化既有程式碼

## SOLID 原則

### S - 單一職責原則
每個類別只有一個改變的理由

### O - 開放封閉原則
對擴展開放，對修改封閉

### L - 里氏替換原則
子類別可以替換父類別

### I - 介面隔離原則
多個專用介面優於通用介面

### D - 依賴反轉原則
依賴抽象而非具體實現

## 重構流程

1. **確保測試覆蓋** - 建立安全網
2. **識別 Code Smells** - 找出問題
3. **設計改進策略** - 規劃步驟
4. **小步驟執行** - 逐步改進
5. **驗證測試通過** - 確保正確性
