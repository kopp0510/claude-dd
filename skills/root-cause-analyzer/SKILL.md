# Root Cause Analyzer

專注於根本原因分析和系統化問題解決的專家技能。

## 核心能力

- **根本原因分析** - 系統性調查找出根本原因
- **模式識別** - 識別重複問題和失敗模式
- **假設測試** - 科學方法除錯
- **最小影響修復** - 解決根本原因不產生副作用
- **預防策略** - 防止類似問題再次發生

## 除錯方法論

### 科學方法

```yaml
1. 觀察: 確切發生了什麼？
   - 錯誤訊息和 stack trace
   - 系統行為和症狀
   - 環境條件
   - 事件時間軸

2. 假設: 可能的原因是什麼？
   - 基於錯誤模式
   - 系統知識
   - 之前類似問題
   - 程式碼分析

3. 預測: 如果假設正確，應該觀察到什麼？

4. 實驗: 測試假設

5. 分析: 評估結果並精煉理解
```

## RCA 框架

1. **問題評估** - 捕捉錯誤細節和環境
2. **資訊收集** - 收集 logs、系統狀態
3. **假設形成** - 建立可測試的理論
4. **調查** - 驗證假設
5. **根本原因識別** - 找出根本原因
6. **解決方案實作** - 最小、針對性的修復
7. **驗證** - 確認修復有效
8. **預防** - 建議防護措施

## 常用工具

```bash
# 系統層級調查
top -p $PID              # CPU 和記憶體
iostat -x 1              # 磁碟 I/O
netstat -tuln            # 網路連線
strace -p $PID           # 系統呼叫追蹤
```
